# 系统架构说明

## 总体目标

本项目围绕 Unitree G1 构建一条混合式动作策略系统链：

```text
动作源数据
  -> 动作资产处理
  -> 基础运动训练与技能动作训练
  -> 策略产物注册
  -> 部署语义恢复与验证
```

## 能力域职责边界

### 1. 动作资产能力域

负责动作数据输入、标准化、资产合法性检查与产物登记。

- 原始动作数据预处理
- 动作资产转换与清洗
- motion 产物命名与登记
- 资产字段一致性检查

### 2. 基础策略能力域

负责基础运动任务入口、训练计划编排、策略信息登记与部署参数追踪。

- 基础任务配置与启动命令
- 训练输出产物登记
- deploy 参数来源追踪
- 基础策略可用性检查

### 3. 技能策略能力域

负责技能动作任务入口、技能策略信息登记与部署交接语义检查。

- 技能任务配置与启动命令
- 技能策略产物登记
- deploy handoff 元数据检查
- obs/action 到命令链语义追踪

## 集成方式

1. **合同层**：记录动作资产、关节顺序、策略产物和部署 handoff。
2. **适配层**：封装 motion/base/skill 能力域入口。
3. **算法层**：保存基础运动与技能动作跟踪训练实现。
4. **注册表**：保存本项目能识别和展示的 motion、task、policy。
5. **pipeline 层**：串联流程，并控制训练、导出和部署检查入口。

## 设计原则

- 能力域之间通过显式合同传递语义，不依赖隐式文件命名。
- 路径默认指向本仓库，便于迁移、测试和交付。
- 高成本训练默认不自动执行，必须由 workflow 配置显式开启。
